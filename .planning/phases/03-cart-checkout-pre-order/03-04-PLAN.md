## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cart core -> serializer pre-Order | Dados internos do cart entram em resposta publica. |
| serializer -> storefront futura | Cliente consome completude mas nao e fonte de verdade. |
| middleware -> logs/Sentry | Response shaping nao deve ampliar logs com PII. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-T | Tampering | `calculateCheckoutDataComplete` | mitigate | Calcular a partir do cart atual, itens positivos, email/endereco validos, variante vendavel e Brasil/BRL. |
| T-03-04-I | Information Disclosure | `serializeStoreCartPreOrder` | mitigate | Omitir/mascarar documento fiscal e remover campos de payment/order/fulfillment da resposta. |
| T-03-04-R | Repudiation | razoes de incompletude | accept | Razoes sao codigos deterministas sem PII; nao representam evento financeiro. |
| T-03-04-E | Elevation of Privilege | middleware cart | mitigate | Middleware apenas serializa resposta; nao altera ownership nem aceita customer do body. |
| T-03-04-B | Tampering | checkout boundary | mitigate | Grep e testes impedem completion/payment/order/webhook/fulfillment. |
</threat_model>
