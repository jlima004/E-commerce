# 10-01 - TrackingAccessToken Contract/Model/Hash/Expiry/Revocation

## Objective

Create the local secure token contract for guest tracking access without
creating the public route yet. Plaintext tokens may be generated and returned
only once from the minting helper; only keyed hashes are persisted.

## Scope

Future execution may add:

- `tracking-access-token` module registration;
- `TrackingAccessToken` model;
- draft migration only, unless real migration is separately approved;
- token generation helper;
- token HMAC hashing helper;
- constant-time hash comparison helper;
- expiry and revocation helpers;
- sanitized metadata/error helpers;
- focused unit tests.

## Model Contract

Planned model fields:

- `id`;
- `order_id`;
- `gelato_fulfillment_id`;
- `token_hash`;
- `status`: `active | expired | revoked`;
- `expires_at`;
- `revoked_at`;
- `last_used_at`;
- `created_for`: safe enum such as `guest_tracking`;
- `created_at`;
- `updated_at`;
- `deleted_at`.

Planned indexes:

- unique `token_hash`;
- index `order_id`;
- index `gelato_fulfillment_id`;
- index `status, expires_at`;

Do not include plaintext token, token prefix, token suffix, raw IP, full
user-agent, full e-mail, phone, CPF/CNPJ, address, payment data, raw Gelato
payload, raw Order payload, headers, cookies, or secrets.

## Hashing Contract

- Generate at least 32 random bytes with `crypto.randomBytes`.
- Encode as URL-safe base64url or equivalent.
- Hash with `HMAC-SHA256(TRACKING_TOKEN_PEPPER, token)`.
- Store only the hash.
- Verify by recomputing candidate hash and using `timingSafeEqual`.
- Fail closed in production-like runtime if `TRACKING_TOKEN_PEPPER` is missing.
