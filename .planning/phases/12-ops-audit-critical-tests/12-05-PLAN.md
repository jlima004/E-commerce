## Data/schema decisions

- IDs RefundRequest/ExchangeRequest são pré-gerados para que intent referencie a futura entidade factual.
- `action_attempt_id` muda por tentativa; correlation/idempotency preservam ligação sem uniqueness.
- AdminActionLog não substitui trails dos módulos; apenas registra actor/intenção/outcome.

## Security/redaction constraints

- Body-supplied operator IDs são rejeitados.
- Snapshots são allowlisted; notes, payload, endereço, documento, token e detalhes Stripe/Gelato não entram.
- API key/missing actor falha antes da mutação; segurança registra somente código sanitizado.

## Test strategy

- HTTP para actor user, actor ausente/API key, sucesso, guard blocked, intent failure, domain failure, falha adicional do append failed, outcome failure e retry.
- HTTP preserva criação por ID pré-gerado, regras Phase 11 e ordem dos collaborators; PostgreSQL real do 12-04 prova append-only.
- Em refund e exchange, outcome audit failure após domínio persistido preserva contrato HTTP de sucesso e prova callback do domínio executado exatamente uma vez.
