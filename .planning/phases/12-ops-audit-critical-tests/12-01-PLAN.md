## Data/schema decisions

- O runner não cria schema de domínio; deixa `medusaIntegrationTestRunner` aplicar em banco temporário as migrations do app, incluindo o draft Gelato materializado sem alteração de DDL.
- O alvo alternativo precisa usar database name prefixado `p12_disposable_`; host externo é rejeitado.
- Nomes/portas/senhas são efêmeros e não persistidos.

## Security/redaction constraints

- Credenciais, DSN, variáveis de ambiente completas e logs do container com segredos não entram em evidência.
- O guard recusa hosts não loopback e nomes de banco não descartáveis.
- Nenhum comando destrutivo pode apontar para uma URL não validada.

## Test strategy

- Unit/TDD no helper para validação de alvo e redaction.
- Integração real via `medusaIntegrationTestRunner` para migrations, constraints e isolamento.
- Falha de infraestrutura é BLOCKED com código próprio, não `skip`.
