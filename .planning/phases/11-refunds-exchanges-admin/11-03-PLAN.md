# 11-03 - ExchangeRequest Admin Workflow And Manual Correios Flow

## Objective

Create the operational exchange workflow for defective or wrong products, with
manual/semi-automatic reverse logistics fields entered by an operator. Exchange
state must stay separate from refund and financial state.

## Scope

Future execution may add:

- `exchange-request` module/model/service;
- draft migration only, unless real migration is separately approved;
- Admin-only create/update routes or commands;
- validated status transitions;
- manual Correios tracking/authorization fields;
- sanitized operator/customer notes;
- focused unit and Admin/API tests.

## Tasks

1. Create `ExchangeRequest` model and status vocabulary.
2. Add draft migration for the model; do not apply it.
3. Register the module only if this slice owns runtime registration and then run
   the required future build gate.
4. Implement Admin create/update commands for `defect` and `wrong_product`.
5. Validate status transitions and affected-item summaries.
6. Add manual reverse-logistics fields:
   - provider enum such as `correios_manual`;
   - reverse tracking code;
   - reverse authorization code;
   - reverse label/reference;
   - return milestone timestamps.
7. Sanitize notes and reject prohibited raw payload/PII/secrets fields.
8. Ensure exchange creation does not create refund, update financial state,
   create Order, or dispatch Gelato.
9. Add tests for statuses, manual fields, sanitization, and scope negatives.

## Verification

Future execution should run focused tests for exchange and Admin routes. If the
slice changes module registration, `env.ts`, or `medusa-config.ts`, it must also
run:

```bash
cd apps/backend && HOME=/tmp XDG_CONFIG_HOME=/tmp TMPDIR=/tmp ADMIN_DISABLED=true npm run build
```

Do not call Correios API. Do not call Gelato. Do not create refund
automatically. Do not start Phase 12.

## Acceptance Criteria

- Operator can create/manage `ExchangeRequest` for defect.
- Operator can create/manage `ExchangeRequest` for wrong product.
- Manual Correios/reverse-logistics codes can be entered and updated.
- Exchange does not create refund automatically.
- Exchange does not update final financial state.
- Exchange does not create Order.
- Exchange does not trigger automatic Gelato dispatch/replacement.
- No Correios API, Gelato real call, Stripe real call, real migration, deploy,
  or Phase 12 work is introduced.
