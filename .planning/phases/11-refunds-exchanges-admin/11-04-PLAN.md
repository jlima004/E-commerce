# 11-04 - Final Validation And Manual Gate

## Objective

Validate the complete Phase 11 refund/exchange scope, document negative proofs,
and stop at the manual gate before Phase 12.

## Required Positive Proofs

1. Refund request rejects zero/negative amount.
2. Refund request rejects amount above locally available captured amount.
3. Refund request rejects currency mismatch.
4. Concurrent/idempotent refunds do not exceed captured amount.
5. Local financial state updates only after trusted Stripe refund webhook
   confirmation.
6. Partial refund recomputes financial state.
7. Total refund recomputes financial state.
8. Exchange request can be created and managed for defect.
9. Exchange request can be created and managed for wrong product.
10. Manual Correios reverse-logistics fields can be entered and updated.

## Required Negative Proofs

1. Refund does not create Order.
2. Refund does not change `order_status` automatically to `canceled`.
3. Total refund does not force `order_status = canceled`.
4. Exchange does not create refund automatically.
5. Exchange does not update financial state.
6. Exchange does not call Correios API real.
7. Exchange does not trigger Gelato dispatch/replacement.
8. No alteration in Gelato dispatch/tracking.
9. No alteration in public tracking.
10. No alteration in Order birth rule.
11. No broad `OperationalAlert` module start.
12. No broad `AdminActionLog` module start.
13. No Stripe CLI smoke real.
14. No real migration applied.
15. No Stripe real call in tests.
16. No Gelato real call.
17. Phase 12 not started.
