## Contrato de scan bruto fail-closed (logs)

Fluxo obrigatório: `bounded raw logs -> raw sensitive scanner -> sensitive_match_count == 0 -> sanitized projection -> delete raw file`. Terminantemente proibido: `raw logs -> sanitized projection -> scan only the projection`.

Objeto JSON único allowlisted emitido pelo scanner (nenhuma outra chave, nenhuma linha/substring/match bruto):

```json
{
  "credential_url_match_count": 0,
  "authorization_match_count": 0,
  "cookie_session_match_count": 0,
  "secret_token_match_count": 0,
  "payment_sensitive_match_count": 0,
  "pii_match_count": 0,
  "sensitive_match_count": 0
}
```

Padrões fechados por categoria (correspondência exige contexto estrutural; proibido casar palavra solta sem contexto, ex. `token`/`secret`/`email`/`payment` isolados, para não gerar falso positivo sobre código já saneado):

- Credential URLs: URL de credencial PostgreSQL; URL de credencial Redis; URLs com userinfo `user:password@host`.
- Authorization: `Authorization:`; `Bearer <token>`; material de Basic auth.
- Cookies/sessões: `Cookie:`; `Set-Cookie:`; IDs de sessão.
- Secrets/tokens: chaves secretas Stripe; webhook secrets; `client_secret`; atribuições de API key; material tipo JWT; credenciais Gelato/Resend/PostHog/Sentry; valores de `DATABASE_URL`/Redis URL/variáveis de provider.
- Payment-sensitive: client secrets do Stripe; QR/copia-e-cola Pix; sequências tipo PAN de cartão; token de tracking em texto puro.
- PII: endereços de e-mail; CPF formatado ou sequência equivalente; telefone brasileiro plausível.

C15 só é PASS antes de qualquer projeção se `sensitive_match_count == 0` E todos os seis contadores individuais forem zero. Qualquer contagem maior que zero é `BLOCKED`: projeção saneada proibida, output do match bruto proibido, `heroku-raw.log` apagado imediatamente, e apenas a recomendação de rollback gate é aberta (rollback nunca executado aqui).

Campos que o SUMMARY pode persistir para este gate — nunca linha ou conteúdo bruto:

```
raw_log_scan_performed: true
raw_log_line_count: <0..200>
credential_url_match_count: 0
authorization_match_count: 0
cookie_session_match_count: 0
secret_token_match_count: 0
payment_sensitive_match_count: 0
pii_match_count: 0
sensitive_match_count: 0
sanitized_projection_created: true
raw_log_file_deleted: true
```

Mais os campos da própria projeção saneada (timestamps arredondados, process type, status, códigos saneados, release/SHA, contagens limitadas).
