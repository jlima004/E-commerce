# Phase 12.1 — Validation Strategy

## Purpose

Este contrato mapeia os 18 closure criteria para planos, tarefas, superfícies, classes de evidência, sanitização e decisões PASS/BLOCKED. Ele não autoriza execução nem declara resultado.

## Evidence classes

Cada item futuro usa exatamente uma classe:

- `local process`
- `local PostgreSQL disposable`
- `Heroku release`
- `production read-only`
- `production mutation`
- `real external provider`
- `inferred only`

Inferência e evidência histórica nunca são promovidas a execução atual.

## Release identity contract (P12_RELEASE_CONTRACT_V1)

Nomenclatura única para C05/C12/C16/C17, exigida de forma idêntica nos SUMMARYs de 12.1-02, 12.1-03 e 12.1-06:

- `predeploy_current_release`: release Heroku ativa imediatamente antes do deploy autorizado do 12.1-03; é o rollback target primário e único.
- `previous_eligible_release`: candidata elegível anterior à `predeploy_current_release`; evidência histórica apenas — nunca substitui automaticamente o rollback target se a primária se tornar inelegível.
- `deployed_release` / `candidate_sha`: release criada pelo deploy autorizado do 12.1-03 via refspec `$P12_CANDIDATE_SHA`:refs/heads/main` e o SHA completo testado no 12.1-01 (`execution_base_sha` propagado pelo 12.1-02).
- `rollback_target_sha`: SHA completo da release `predeploy_current_release` para a qual um rollback documentado apontaria; se o rollback target se tornar inelegível, o resultado é `BLOCKED` com nova decisão humana e novo preflight — nenhuma substituição aproximada por `previous_eligible_release` é permitida.
- `documentary_head`: HEAD Git no momento do deploy; pode ser descendente allowlisted do `candidate_sha` somente por commits dos SUMMARYs 01/02; nunca é promovido a release SHA.

C17 exige compatibilidade SHA-based entre `rollback_target_sha` e `candidate_sha` (schema/migration compatibility) e cross-check de que os três SUMMARYs (12.1-02, 12.1-03, 12.1-06) usam os mesmos IDs/SHA para cada papel. C05 exige `deployed_sha == candidate_sha` e proíbe deploy do HEAD documental. Qualquer divergência de nomenclatura ou de valor entre os três SUMMARYs bloqueia C17.
